hexo + github 个人博客使用

发表于 2020-06-10 分类于 技术杂文 阅读次数： Valine：

hexo 是一个快速、简洁且高效的博客框架，采用 Node.js，使得网页资源的渲染更加快速。Hexo 支持 GitHub Flavored Markdown 的所有功能，甚至可以整合 Octopress 的大多数插件。

一、相关指令学习

1. init
$ hexo init [folder]
新建一个网站。如果没有设置 folder ，Hexo 默认在目前的文件夹建立网站。

2. new
$ hexo new [layout] <title>

参数 描述
layout page所使用的模板
title page的标题
-p/–path 自定义新文章的路径
-r/–replace 如果存在同名文章，将其替换
-s/–slug 文章的Slug，作为新文章的文件名和发布后的URL

$ hexo new page --path about/me "About me"
创建一个 source/about/me.md 文件，同时 Front Matter 中的 title 为 “About me”

新建一篇文章。如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数代替。如果标题包含空格的话，请使用引号括起来。

3. generate
$ hexo generate（也可写做$ hexo g）
生成静态文件

参数 描述
-d/–deploy 文件生成后立即部署网站
-w/–watch 监视文件变动
-b/–bail 生成过程中如果发生任何未处理的异常则抛出异常
-f/–force 强制重新生成文件。Hexo引入了差分机制，如果 public 目录存在，那么 hexo g 只会重新生成改动的文件。使用该参数的效果接近 hexo clean && hexo generate
-c/–concurrency 最大同时生成文件的数量，默认无限制

4. publish
$ hexo publish [layout] <filename>
发表草稿

5. server
$ hexo server
启动服务器。默认情况下，访问网址为：http://localhost:400/ 。

参数 描述
-p/–port 重设端口
-s/–static 只使用静态文件
-l/–log 启动日记记录，使用覆盖记录格式

6. deploy
$ hexo deploy（可写做 $ hexo d）
部署网站
参数|描述
:–:|:-:
-g/–generate|部署之前预先生成静态文件

写好文章后可以直接输入：
$ hexo d -g
进行部署并生成静态文件
7. clean
$ hexo clean
清除缓存文件 （db.json）和已生成的静态文件 （public）.
在某些情况（尤其是更换主题后），如果发现您对站点的更改无论如何也不生效，您可能需要运行该命令
